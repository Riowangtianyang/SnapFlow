## 1. 项目概述

### 1.1 核心功能

无代码可视化爬虫工具，用户通过截图演示爬虫流程，AI Agent 自动理解意图并生成可批量执行的工作流。

**核心范式**：用户演示一次，Agent 自动执行批量任务。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| 简单易用 | 用户只需上传截图、标注意图，无需编码 |
| 智能生成 | AI 自动理解语义，生成工作流 |
| 可微调 | 用户可修改意图、编辑工作流 |
| 可扩展 | 插件系统支持知识库、反爬虫、搜索等扩展 |
| 易于分享 | JSON + 画布双视图，适合 GitHub 技术展示 |

### 1.3 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + TypeScript | 生态成熟 |
| 前端画布 | React Flow | 节点编排 |
| 前端状态 | Zustand | 轻量状态管理 |
| 前端样式 | Tailwind CSS | 快速开发 |
| 后端 | FastAPI | 异步、高性能、自动文档 |
| 浏览器 | Playwright (Chrome) | CDP 支持好 |
| 数据库 | SQLite | 零配置、个人使用 |
| 任务队列 | 内存队列 + 可扩展 | 轻量设计 |

## 7. AI 意图理解

### 7.1 意图生成流程

```
用户上传截图 + URL
       │
       ▼
  ┌─────────────┐
  │ 截图 + URL  │
  │   发给 AI   │
  └──────┬──────┘
         ▼
  ┌─────────────────┐
  │ AI 分析意图     │
  │ - 单步意图      │
  │ - 总意图        │
  │ - 循环/翻页推断 │
  └──────┬──────────┘
         ▼
  ┌─────────────────┐
  │ 意图模糊?       │
  └──────┬──────────┘
         │
    ┌────┴────┐
    │  是     │  否
    ▼         ▼
┌────────┐  ┌──────────────┐
│ 反问   │  │ 生成工作流   │
│ 用户   │  └──────┬───────┘
└────────┘         ▼
                   │
              ┌────┴────┐
              │ 复杂?   │
              └────┬────┘
                   │
              ┌────┴────┐
              │  是     │
              ▼         ▼
         ┌────────┐  ┌──────────┐
         │ JSON编辑│  │ 直接运行 │
         │ 用户微调│  └──────────┘
         └────────┘
```

### 7.2 AI 反问场景

| 场景 | AI 反问 |
|------|---------|
| 意图模糊 | "您说的'所有政策'是指当前页面的还是整个网站的？" |
| 翻页不确定 | "这个列表大概有多少页？或者要爬取到什么时候？" |
| 循环边界不清 | "需要提取前几条数据？还是全部？" |

### 7.3 循环/翻页识别

```python
# AI 识别的模式
LOOP_PATTERNS = {
    "all": ["所有", "全部", "每一个", "所有的"],
    "firstN": ["前", "前几条", "最新的"],
    "dateRange": ["最近一周", "今天", "本周", "本月"]
}

PAGINATION_PATTERNS = {
    "fixed": ["第1页到第10页", "前5页"],
    "untilEmpty": ["直到没有更多", "翻完所有"],
    "loadMore": ["下拉加载更多", "点击加载更多"]
}
```

### 11.2 设计决策记录

| # | 决策 | 原因 |
|---|------|------|
| 1 | 使用 React Flow 作为画布 | 生态成熟，定制灵活 |

| 3 | AI 生成意图，用户可微调 | 减少用户操作，AI 理解语义 |

| 5 | 插件系统支持扩展 | 未来可接入知识库、反爬虫等 |
| 6 | 异常时显示原因 | 用户体验优化，便于排查问题 |
